Help & Docs

// How to use Backupman effectively

1. How to use JSON Settings Files

Backupman uses a JSON file to save all your schedules, history, and destinations. You can Create New Setting to start fresh, or Export Settings to save your configuration and move it to another PC. Importing a setting file will overwrite your active data!

2. Specifying Multiple Destinations

You can add multiple backup destinations to a single schedule! Backupman will sequentially copy your source data to all listed destinations. Click the "Add Destination" button inside the Schedule editor to add another location (such as an external drive AND a network share).

3. Output Naming Conventions

When configuring a destination, you can use Name Templates to dynamically name your backup files or folders. For example:
{name} : The original name of the source (e.g. "my_data")
{date} : Current date (e.g. "2026-03-19")
{time} : Current time (e.g. "14-30-00")
Template example: {name}_{date}_{time} becomes my_data_2026-03-19_14-30-00.bak

4. Common Mistake: Folder Paths

When backing up a folder, Backupman creates a new folder/file at the destination exactly as your template dictates.
Example Scenario: You have a folder test/files and you want it backed up inside test_backup/files.
❌ Incorrect Way: Setting destination path to test_backup/files and output name to files. This will result in test_backup/files/files!
✅ Correct Way: Set destination path to test_backup and output name to files. Backupman will look at test_backup, create a folder named files, and then perfectly copy your data inside, giving you test_backup/files!
